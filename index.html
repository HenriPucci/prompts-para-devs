#D01 — Análise de Stack Trace / Stack Trace Analysis
Analise este stack trace e explique:

1) Qual é a causa raiz do erro.
2) Em qual camada da aplicação ele ocorre (controller, service, repository).
3) Sugira 3 possíveis correções ordenadas por probabilidade de resolver o problema.

Stack trace: [COLE AQUI]
💡 Dica: Sempre cole o stack trace completo, não apenas a última linha.
#T03 — TDD: Desenvolver com Teste Primeiro
Quero desenvolver [FUNCIONALIDADE] usando TDD. Guie-me passo a passo:

1) Escreva o PRIMEIRO teste (que vai falhar).
2) Implemente o MÍNIMO de código para passar.
3) Refatore.
4) Repita para o próximo requisito.

Requisitos: [LISTE OS REQUISITOS]
